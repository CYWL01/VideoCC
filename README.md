# 视频字幕自动匹配工具

一个 Windows 桌面工具，用于扫描视频文件和字幕文件，自动根据文件名中的剧名、季数、集数进行智能匹配，并批量复制、移动或重命名字幕。

## 功能总览

- 自动扫描视频文件夹和字幕文件夹，支持递归扫描子文件夹。
- 自动识别文件名和文件夹里的剧名、季数和集数，例如 `Show.Name.S01E05.mkv`、`电视剧一\第一季\01.mkv`、`[01]`、`第12集`。
- 自动按"文件名剧名优先 → 文件夹上下文兜底 → 季数 → 集数"匹配视频和字幕，避免多部剧同集数串匹配。
- 如果视频和字幕文件名都含有剧名但互相冲突，即使路径相同也不会强行匹配。
- 结果按 **剧名 → 季数/文件夹排序 → 集数** 三级升序排序。季数优先使用显式关键词（Season、S、季），无显式关键词时使用文件夹数字前缀（仅用于排序，不影响匹配）。
- 表格列宽自适应最佳列宽，根据内容自动调整宽度。
- 自动排除非剧集文件（NCOP、NCED、PV、Menu、Sample、Trailer 等），避免误识别。
- 支持联网翻译功能：开启后遇到中/日/韩文等非英文剧名自动翻译成英文后匹配。
- 支持手动匹配，可通过下拉列表选择字幕，也可直接浏览选择任意字幕文件。
- 支持勾选部分匹配项后批量处理。
- 支持三种字幕处理方式：复制字幕、移动字幕、复制并重命名为视频文件名。
- 支持撤销本次运行期间产生的文件操作（复制/移动/重命名均可撤销）。
- 内置使用说明窗口，提供格式说明、使用步骤、常见问题和处理建议。
- 内置许可窗口，以渲染后的 Markdown 格式显示许可协议。
- 所有扫描和文件操作均在本机完成；翻译功能可选联网。

## 支持格式

### 视频格式

`mp4`、`mkv`、`avi`、`wmv`、`mov`、`flv`、`webm`

### 字幕格式

`srt`、`ass`、`ssa`、`sub`、`sup`、`txt`

## 集数识别规则

软件会从文件名中提取集数，提取成功后会统一成 `E01` 或 `S01E01` 这样的内部匹配编号。

| 类型 | 示例 | 识别结果 |
|---|---|---|
| 标准季集 | `Show.Name.S01E05.1080p.mkv`、`Show.S01.E05.mkv`、`Show.S1_E5.mkv` | `S01E05` |
| Season/Episode | `Season 02 Episode 12.srt`、`Season 2 Ep.12.ass` | `S02E12` |
| E/Ep/Episode 前缀 | `Anime.E03.mkv`、`Anime EP.12.srt`、`Episode 8.ass` | `E03`、`E12`、`E08` |
| 中文集数 | `某剧第12集.mp4` | `E12` |
| 中文话数 | `动画第05话.ass` | `E05` |
| 方括号 | `[VCB-Studio] Title [01][1080p].mkv` | `E01` |
| 连字符 | `Anime-01-1080p.mkv` | `E01` |
| 下划线 | `Anime_01.zh-CN.ass` | `E01` |
| 空格加点 | `Series 01.1080p.mkv` | `E01` |
| 点号包裹 | `Series.01.1080p.mkv` | `E01` |
| 数字开头 | `01-Anime.mkv` | `E01` |
| 数字结尾 | `Anime-01.srt` | `E01` |
| 纯数字 | `1.mkv`、`01.ass`、`001.srt` | `E01` |

## 自动匹配逻辑

软件采用以下顺序进行自动匹配：

1. 先扫描视频文件，提取每个视频的文件名剧名、路径剧名、季数和集数。
2. 再扫描字幕文件，提取每个字幕的文件名剧名、路径剧名、季数和集数。视频和字幕目录的扫描同时并行进行。
3. 文件名剧名优先级最高。如果双方文件名都能识别出剧名，则只用文件名剧名判断是否同剧，路径不会覆盖文件名冲突。
4. 文件名缺少剧名时，才用文件夹路径作为兜底。例如 `电视剧一\第一季\01.mkv` 会从文件夹得到剧名 `电视剧一`、从 `第一季` 得到季数 `S01`，从 `01.mkv` 得到集数 `E01`。
5. 自动匹配时必须先满足集数相同；`1`、`01`、`001` 都会归一到同一集 `E01`。
6. 如果两边都有季数，季数也必须相同。如果一边没有季数但存在多季歧义，软件不会自动选择。
7. 如果字幕没有剧名信息，但当前同一集没有多部剧或多季歧义，软件允许按集数兜底匹配。
8. 如果视频和字幕文件名都含有剧名但互不相同（如中文名 vs 英文名），只要当前同一集没有其他不同剧名的视频，软件仍会按集数自动匹配。
9. 开启联网翻译后，对于中/日/韩文等非英文剧名，软件会先调用免费翻译服务将剧名翻译成英文，再重新比较剧名是否一致。翻译结果会缓存，同一文本只请求一次。
10. 如果存在同集多剧、同集多季、多个字幕同分等歧义，软件不会强行自动选择，需在"手动匹配"页选择。
11. 如果文件名和路径都无法提取集数，该行不会自动匹配字幕，但仍可手动选择字幕。
12. 匹配完成后，结果按"剧名 → 季数/文件夹排序 → 集数"三级升序排序。季数优先使用显式关键词（Season、S、季），没有时用文件夹数字前缀（如 `01-...` 为 1），用于排序但不影响匹配。
13. 表头固定不动，滚动内容时表头始终可见。

### 非剧集自动排除

在提取集数之前，软件会先检查文件名是否包含以下非剧集标记。如果匹配则跳过该文件（不提取集数、不参与自动匹配）：

`NCOP`、`NCED`、`PV`、`PVED`、`MENU`、`CM`、`SPECIAL`、`TRAILER`、`PREVIEW`、`SAMPLE`

例如 `[VCB-Studio] Title [NCED_EP12][Ma10p_1080p][x265_flac].mkv` 会被自动跳过，不会误识别为第 12 集。

## 使用流程

1. 在“视频”一行点击“浏览”，选择视频所在文件夹。
2. 在“字幕”一行点击“浏览”，选择字幕所在文件夹。
3. （可选）如果视频和字幕文件名语言不同（如中文剧名 vs 英文剧名），点击“🌐 翻译关”按钮切换为绿色高亮的“🌐 翻译开”，开启联网翻译辅助匹配。
4. 点击“扫描”，软件会读取支持格式的视频和字幕，并尝试按文件名剧名、路径上下文、季数、集数自动匹配。
5. 在“自动匹配”页检查结果。
6. 如有不准确的项目，切换到“手动匹配”页，通过下拉框或 `...` 按钮指定正确字幕。
7. 勾选需要处理的项目。
8. 在顶部选择操作方式（复制字幕、移动字幕、重命名）。
9. 点击“执行”完成文件处理。
10. 若需要撤回本次操作，点击“撤销”。

## 操作方式说明

| 操作 | 效果 | 适合场景 |
|---|---|---|
| 复制字幕 | 将字幕复制到视频所在目录，保留原字幕 | 想保留原字幕文件 |
| 移动字幕 | 将字幕移动到视频所在目录，原位置不再保留 | 想整理字幕文件位置 |
| 重命名 | 复制字幕到视频目录，并改成与视频相同的文件名 | 推荐给播放器自动加载字幕 |

重命名示例：

```text
视频：D:\Media\Anime\Title.E01.mkv
字幕：D:\Subs\Title.01.zh-CN.ass

执行“重命名”后：
D:\Media\Anime\Title.E01.ass
```

## 全部按钮详细说明

### 顶部路径选择

**浏览（视频/字幕各一个）**
- 点击后弹出 Windows 文件夹选择窗口。
- 选择后路径自动填入文本框。
- 软件不校验路径是否真实存在，扫描时才会检查。

### 顶部操作模式

**复制字幕（默认选中）**
- 高亮蓝色表示当前选中状态。
- 执行时：将字幕文件复制到视频所在目录，原字幕文件保留在原位置不动。
- 目标文件名 = 原字幕文件名（保持不变）。

**移动字幕**
- 点击切换为蓝色高亮，取消复制/重命名的选中。
- 执行时：将字幕文件移动到视频所在目录，原位置不再保留该文件。
- 目标文件名 = 原字幕文件名。

**重命名**
- 点击切换为蓝色高亮，取消复制/移动的选中。
- 执行时：将字幕文件复制到视频所在目录（保留原文件），并将文件名改为与视频文件同名。
- 目标文件名 = 视频文件名 + 字幕扩展名。例如 `视频.mkv` → `视频.ass`。
- 推荐给 PotPlayer、MPC-HC 等播放器自动加载字幕使用。

### 右侧功能按钮

**扫描**
- 前置条件：视频和字幕路径都必须非空且路径真实存在，否则弹出提示。
- 点击后执行以下步骤：
  1. 清除所有内部缓存（正则解析缓存、剧名标准化缓存）。
  2. 递归扫描视频文件夹，找出所有支持的视频文件。
  3. 递归扫描字幕文件夹，找出所有支持的字幕文件。
   4. 为每个文件提取：文件名剧名、文件夹路径剧名、季数、集数。文件夹数字前缀（如 `01-...` 中的 01）提取为排序值，仅用于排序，不影响季数匹配。
   5. 自动排除非剧集文件（NCOP、NCED、PV、Menu、Sample、Trailer 等）。
   6. 根据剧名、季数、集数进行自动匹配。
   7. 结果按 **剧名 → 季数/文件夹排序 → 集数** 三级升序排序。季数优先使用显式关键词（Season、S、季），无显式关键词时使用文件夹数字前缀，仅用于排序不影响匹配。
   8. 清空之前的撤销记录。
   9. 扫描完成后，标签页右侧状态栏永久显示详细结果（视频/字幕数量、已匹配/未匹配数、用时），按钮上短暂显示简要结果后恢复，不会弹出阻塞窗口。

**“▶️ 执行”**
- 前置条件：至少有一行已勾选且该行已选择了字幕路径，否则弹出“没有选中的匹配项”。
- 点击后根据当前选中的操作模式（复制/移动/重命名）处理所有符合条件的勾选项：
  - 只处理 **已勾选（IsSelected=true）且已设置字幕路径（SubtitlePath非空）** 的项目。
  - 跳过未勾选或未设置字幕的项目。
- **执行时如果目标文件已存在，会直接覆盖**（不会提示确认）。
- 执行成功后，该操作会记录到撤销列表，供后续撤销使用。
- **每次执行会清空之前的撤销记录**，即撤销只作用于最近一次执行。

**“↩️ 撤销”**
- 前置条件：本运行期间至少执行过一次操作且有未撤销的记录，否则弹出“没有可撤销的操作”。
- 撤销机制与边缘情况：

  | 场景 | 执行的操作 | 撤销的效果 | 特殊情况 |
  |---|---|---|---|
  |① | 选择了**复制字幕**并执行 | 删除复制生成的文件。原字幕文件保留在原位置不动。 | 如果生成的文件已被手动删除，撤销会跳过该项 |
  |② | 选择了**移动字幕**并执行 | 将生成的文件移动回原位置。视频目录下不再保留该文件。 | 如果原位置已有同名文件（用户手动放入），撤销会覆盖该文件 |
  |③ | 选择了**重命名**并执行 | 删除生成的重命名文件。原字幕文件保留在原位置不动。 | 同上，生成文件已删除则跳过 |
  |④ | 执行后发现字幕选错了，重新选择后再次执行 | 第二次执行会清空第一次的撤销记录。**只能撤销第二次执行的操作** | 提示：每次执行前请确认操作方式正确 |
  |⑤ | 撤销时某项失败（文件被其他程序占用等） | 成功的项正常撤销，失败的项保留在撤销列表中 | 再次点击撤销可以继续尝试撤销失败的项 |
  |⑥ | 关闭软件后再次打开 | 撤销记录不保存，**无法撤销之前运行期间的操作** | — |

- 撤销只会删除或移动本软件生成的文件，不会删除或修改用户的原始字幕文件（移动操作撤销时，原始文件移回原位置）。

**“🌐 翻译关/开”**
- 点击切换开关状态。
- **关闭状态**（灰色🌐翻译关）：不进行联网翻译，剧名冲突时只用本地规则（按集数兜底匹配/按目录名匹配）。
- **开启状态**（绿色🌐翻译开）：遇到中/日/韩文等非英文剧名时，自动调用免费的翻译服务将剧名翻译成英文，再与另一方比较。翻译结果会缓存，同一文本只请求一次。
- 默认免费服务地址：`https://libretranslate.de/translate` 和 `https://translate.faucet.dev/translate`。
- 如果网络不通或者服务超时，翻译自动跳过，不影响普通匹配流程。
- 此开关状态会同步显示在“⚙️ 配置”窗口中。

**“⚙️ 配置”**
- 点击打开翻译服务配置窗口。
- 窗口内容：
  - **当前状态**：显示翻译功能的开/关状态（与主界面按钮同步）、当前使用的 API 地址。
  - **自定义 API 服务器**：输入框可填入自己的翻译服务器地址。留空则使用默认的免费服务器。
  - **保存**：保存填写的 API 地址。
  - **恢复默认**：恢复为内置的免费服务器。
  - **测试连接**：异步测试输入的地址是否可用，会返回翻译结果预览。
  - **详细说明**：什么是翻译 API、默认情况不需要任何操作、如何自己搭建翻译服务器（12步教程）、公共服务器列表。

**“📖 说明”**
- 点击打开使用说明窗口，以卡片式布局显示完整的格式说明、使用步骤、常见问题和处理建议。

**“ℹ️ 许可”**
- 点击打开许可协议窗口。
- 从软件目录加载 `LICENSE` 文件，以渲染后的 Markdown 格式展示。
- 如果 `LICENSE` 文件不存在，显示内置的备用许可文本。

### 中间表格区域

**标签页切换与状态栏**
- 顶部两个按钮切换“⚡ 自动匹配”和“🔧 手动匹配”视图。
- 标签页右侧显示扫描状态：视频/字幕数量、已匹配/未匹配数、用时。

**“⚡ 自动匹配” 标签页**
- 显示所有视频文件及其自动匹配到的字幕。
- 表头固定不动，滚动内容时表头始终可见。
- 列：`#` 序号 → `📹 视频文件` → `集数` → `📜 匹配字幕` → `集数` → `✓`。
- 右侧 `✓` 勾选框：默认全选，可手动取消。执行时只处理已勾选的项目。
- 未匹配的行显示“— 未匹配 —”。
- 鼠标滚轮可滚动内容。

**“🔧 手动匹配” 标签页**
- 与自动匹配相同的数据，切换到下拉选择模式。
- **下拉框**：列出所有扫描到的字幕文件，选择后自动勾选该行。
- **“...” 按钮**：弹出文件选择窗口，可选择不在扫描结果中的字幕文件。
- 手动匹配页的选择会自动同步回自动匹配页。

## 示例场景

### 示例 1：标准美剧命名

```text
视频：Show.Name.S01E05.1080p.WEB-DL.x265.mkv
字幕：Show.Name.S01E05.Chinese.srt
结果：自动识别为 S01E05，直接匹配成功
```

### 示例 2：季集之间有点号或下划线

```text
视频：Show.S01.E05.mkv
字幕：Show.S1_E5.ass
识别：两者都识别为 S01E05
结果：自动匹配成功
```

### 示例 3：Season / Episode 格式

```text
视频：Drama Season 02 Episode 12.mkv
字幕：Drama Season 2 Ep.12.zh.srt
识别：两者都识别为 S02E12
结果：自动匹配成功
```

### 示例 4：动漫方括号命名

```text
视频：[VCB-Studio] Title [01][Ma10p_1080p][x265_flac].mkv
字幕：[VCB-Studio] Title [01][Ma10p_1080p][x265_flac].SC.ass
结果：方括号元数据 `[VCB-Studio]`、`[Ma10p_1080p]`、`[x265_flac]` 被自动清除，
     剧名统一为 `Title`，集数 `[01]` 被提取为 E01，匹配成功
```

### 示例 5：中文集数命名

```text
视频：某动漫第05集1080p.mp4
字幕：某动漫第05集简中.srt
结果：自动识别为 E05，直接匹配成功
```

### 示例 6：纯数字文件名

```text
视频：01.mkv
字幕：01.ass
识别：两者都识别为 E01
结果：自动匹配成功
```

### 示例 7：视频和字幕前缀不同

```text
视频：Episode_03.mp4
字幕：简体中文.03.ass
结果：都识别为 E03，可自动匹配
```

### 示例 8：多部剧混在一起

```text
视频：ShowA.S01E01.mkv、ShowB.S01E01.mkv
字幕：ShowA.01.srt、ShowB.01.srt
结果：先匹配 ShowA / ShowB，再匹配 E01，不会只按第 1 集串到一起
```

### 示例 9：文件名没有剧名，但文件夹有剧名和季数

```text
视频：
D:\Videos\电视剧一\第一季\01.mkv
D:\Videos\电视剧二\第一季\01.mkv

字幕：
D:\Subs\电视剧一\S01\01.ass
D:\Subs\电视剧二\Season 1\01.srt

结果：
电视剧一\第一季\01.mkv -> 电视剧一\S01\01.ass
电视剧二\第一季\01.mkv -> 电视剧二\Season 1\01.srt
```

### 示例 10：文件名正确但路径放错

```text
视频：D:\Videos\放错的视频目录\ShowA.S01E01.mkv
字幕：D:\Subs\放错的字幕目录\ShowA.01.srt
结果：优先按文件名 ShowA + E01 匹配，路径目录名不会覆盖文件名
```

### 示例 11：文件名冲突时不强行匹配

```text
视频：D:\Videos\ShowA\ShowA.01.mkv
字幕：D:\Subs\ShowA\ShowB.01.srt
结果：视频文件名是 ShowA，字幕文件名是 ShowB，虽然路径都像 ShowA，但不会自动匹配
```

### 示例 12：视频纯数字，字幕文件名信息完整


## 当前实现限制

- 文件名剧名冲突时，路径不会覆盖文件名判断；这种情况需要手动确认。
- 同一集存在多个字幕时，自动匹配不会自动判断哪个语言或版本更合适。
- 执行文件操作前不会显示二次确认弹窗，请先确认勾选项和操作方式。
- 路径不会自动保存，重新打开软件后需要重新选择文件夹。

## 注意事项

- 撤销功能只记录当前软件运行期间的操作，关闭软件后无法撤销。
- 扫描会递归读取子文件夹，如果文件很多，扫描时间会随文件数量增加。
- 自动匹配会优先比较文件名里的剧名，再使用文件夹上下文兜底，然后比较季数和集数；如果剧名信息缺失且同集多剧或多季存在歧义，软件会避免自动选择，需要手动匹配。
- 视频和字幕文件名剧名不同时（如中文名 vs 英文名），只要同一集没有其他剧名的视频冲突，仍可自动匹配。开启联网翻译后，非英文剧名会被翻译成英文后再比较，进一步提升匹配率。
- 联网翻译功能（"🌐 翻译开"按钮）需要网络连接，使用免费的 LibreTranslate 服务。如果网络不可用或服务响应超时，翻译会自动跳过，不影响普通匹配流程。
- 文件处理会覆盖目标路径下同名文件，执行前请确认目标目录中没有需要保留的同名字幕。
- 路径不会自动保存，重新打开软件后需要重新选择文件夹。
- 递归扫描到相同文件名的字幕时，下拉列表会显示相对路径；仍不确定时可使用 `...` 按钮精确选择。
- 不确定结果时，建议先勾选少量项目测试，再批量处理。

## 常见问题

### 为什么有些文件没有自动匹配？

可能原因：
- 文件名中没有可识别的集数（如纯英文标题不含数字）
- 文件名包含非剧集标记（NCOP、NCED、PV 等）被自动跳过
- 同一集存在多部剧或有多个同分字幕，软件避免强行匹配
- 视频和字幕的剧名在文件名层面冲突且无法通过路径或翻译调和

遇到未匹配的项目可在"手动匹配"页手动选择字幕。

### 匹配结果里的排序是怎样的？

结果按 **剧名 → 季数/文件夹排序 → 集数** 三级升序排序。季数优先使用显式关键词（Season、S、季），没有时使用文件夹数字前缀（如 `01-...` 中的 01 作为排序值），仅用于排序不影响匹配。

### 为什么有的文件夹被跳过了？

以纯数字开头的文件夹（如 `11-[VCB-Studio] Title`）会被识别为光盘文件夹自动跳过，避免将光盘编号误判为剧集信息。

### 方括号和文件后缀里的元数据会被清除吗？

会。软件在提取剧名前会清除所有方括号内容（如 `[VCB-Studio]`、`[1080p]`、`[x265_flac]`）和尾部以点号开头的后缀标签（如 `.HKG&X2-sc`），确保剧名不被这些元数据干扰。

### 什么是文件夹排序值？

当文件名中没有显式季数关键词（Season、S、季）时，如果文件夹以数字开头（如 `01-电视剧名`），该数字会被提取为排序值用于排序，但**不会**用于季数匹配。这解决了 `02-...` 文件夹被误判为第二季的问题。

### 扫描过程为什么很快？

扫描同时对视频目录和字幕目录进行并行扫描，并在每个目录内使用多线程并行解析文件信息。

## 内部处理规则

软件在提取剧名时依次执行以下清理步骤，确保匹配准确性：

1. **通用目录名过滤**：跳过名为 "字幕"、"video"、"downloads" 等通用目录名，避免将其作为剧名。
2. **方括号元数据清除**：移除文件名中所有 `[...]` 内容（组名、画质、编码等）。
3. **尾部后缀标签清除**：移除文件名尾部以点号开头的后缀标签（如 `.HKG&X2-sc`）。
4. **通用标记过滤**：从剧名中移除 1080p、x264、HEVC、WEB-DL 等通用技术标记。
5. **光盘文件夹跳过**：以纯数字开头的文件夹（如 `11-[VCB-Studio]`）跳过处理，避免光盘编号被误识别。
6. **中文数字解析**：支持"第三季"→ S03、"第十二话"→ E12 等中文数字季/集。
